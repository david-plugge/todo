# Svelte / Tailwind conventions

- **Svelte 5 runes** (`$state`, `$derived`, `$props`, `$effect`) — never `export let` or the
  legacy store-in-component patterns.
- **Styling is token utilities only.** Design tokens in `src/app.css` map into Tailwind v4
  `@theme`; use `bg-bg`, `text-dim`, `border-line`, `rounded-card`, etc. Do **not** use `dark:`
  variants (tokens flip with `prefers-color-scheme` automatically) or Tailwind's default color
  palette (it's disabled on purpose).
- `src/lib/components/ui/**` is generated by the **shadcn-svelte** CLI — add or regenerate
  components via the CLI, don't hand-edit them. App-specific code lives in `src/lib/*.svelte`.
